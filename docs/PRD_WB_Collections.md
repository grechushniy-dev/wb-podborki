# PRD — Раздел «Подборки»
### Рекламный кабинет WB Инфлюенс

**Версия:** 1.0 | **Дата:** Февраль 2026  
**UI Framework:** Ant Design 5.x | React 18+ | TypeScript

---

## 1. Обзор продукта и цели

Раздел «Подборки» — новый инструмент в рекламном кабинете WB Инфлюенс, позволяющий селлерам участвовать в тематических товарных подборках, которые создают и публикуют блогеры. Раздел встраивается как отдельная вкладка рядом с «Оплата за заказ (CPO)» и «Оплата за пост (CPP)».

**Бизнес-цели:**
- Предоставить селлерам новый канал продвижения товаров через тематические подборки блогеров
- Автоматизировать процесс подачи, одобрения и подтверждения участия
- Повысить вовлечённость блогеров и селлеров в единой экосистеме WB Инфлюенс

**Ключевая проблема:**  
Сегодня попадание в подборки блогеров — неформальный, ручной и непрозрачный процесс. Селлер не видит доступных возможностей, не управляет заявками и не получает структурированной обратной связи.

---

## 2. Целевая аудитория

| Роль | Описание | Ключевые потребности |
|------|----------|----------------------|
| **Селлер** (покупатель слота) | Продавец на WB, зарегистрированный в WB Инфлюенс | Найти релевантные подборки, быстро подать заявку, отслеживать статус |
| **Блогер** (создатель подборки) | Автор контента, создающий подборки в своём кабинете | Получать качественные заявки, одобрять/отклонять участников |
| **Модератор платформы** | Сотрудник WB Инфлюенс | Модерировать подборки перед публикацией в кабинете |

---

## 3. Пользовательский флоу (User Flow)

### Полный жизненный цикл подборки

1. Платформа формирует тематику подборки с описанием
2. Формируется список блогеров, подходящих под тематику
3. Блогеру предлагается создать подборку с минимальной ценой за участие
4. Блогер принимает задание, заполняет сценарий, категории, даты, кол-во товаров
5. Модератор проверяет подборку и публикует её в рекламном кабинете
6. Селлер видит подборку в разделе «Подборки», открывает детали в Drawer (сайд-панель)
7. Селлер выбирает условия (цена, подарок, товары-НМ) и подаёт заявку
8. Блогер **вручную** одобряет или отклоняет каждую заявку
9. При одобрении — селлер получает уведомление и **финально подтверждает** участие
10. Селлер отправляет товар блогеру (бесплатно для селлера)

### Жизненный цикл статусов заявки

```
Ожидает одобрения → Одобрена блогером → Подтверждена → Товар отправлен → Опубликовано
                                       ↘ Отклонена
```

| Статус | Ant Design Tag | Описание |
|--------|---------------|----------|
| Ожидает одобрения | `Tag color='processing'` | Заявка подана, ждёт решения блогера |
| Одобрена блогером | `Tag color='warning'` | Блогер одобрил, ждёт финального подтверждения |
| Подтверждена | `Tag color='success'` | Селлер финально подтвердил участие |
| Товар отправлен | `Tag color='cyan'` | Селлер подтвердил отправку товара |
| Опубликовано | `Tag color='green'` | Подборка вышла в публичный доступ |
| Отклонена | `Tag color='error'` | Блогер отклонил заявку |

## 4. Функциональные требования

### 4.1 Список подборок (главный экран раздела)

**Описание:**  
Главный экран раздела. Навигационная вкладка «Подборки» размещается рядом с CPO и CPP. Селлер видит все активные подборки в виде таблицы с возможностью фильтрации по тематике.

**UI-компоненты Ant Design:**
- `Tabs` — переключение CPO / CPP / Подборки; `Badge` со счётчиком новых подборок
- `Table` — основной список с сортировкой по цене, датам, слотам
- `Select (mode='multiple')` — фильтр по тематикам
- `Input.Search` — поиск по названию подборки
- `DatePicker.RangePicker` — фильтр по датам публикации
- `Tag` — статус подборки (Активна / Набор закрыт)
- `Empty` — состояние при отсутствии результатов
- `Skeleton` — состояние загрузки

**Поля таблицы:**

| Поле | Тип данных | Описание |
|------|-----------|----------|
| Название | `string` | Краткое название подборки |
| Тематика | `string (Tag)` | Категория — используется как фильтр |
| Блогер | `string + Avatar` | Имя блогера с аватаром |
| Цена за слот | `number (RUB)` | Минимальная стоимость участия |
| Свободных мест | `number` | Оставшееся кол-во слотов |
| Дата публикации | `date` | Когда выйдет подборка |
| Статус | `enum (Tag)` | Активна / Набор закрыт / Архив |

**Acceptance Criteria:**
- Список загружается при переходе на вкладку; `Skeleton` отображается во время загрузки
- Фильтр по тематикам работает без перезагрузки страницы
- Подборки с истёкшим дедлайном отображаются со статусом «Набор закрыт»
- Клик на строку открывает `Drawer` с деталями подборки

---

### 4.2 Детальная карточка подборки (Drawer / Сайд-панель)

**Описание:**  
По клику на подборку открывается `Drawer` справа (`placement='right'`, `width=480px`). Список подборок остаётся видимым за панелью. Закрывается крестиком или кликом вне панели.

**UI-компоненты Ant Design:**
- `Drawer` — контейнер, `placement='right'`, `width=480`
- `Descriptions` — блок мета-информации о подборке
- `Avatar + Typography.Link` — блок блогера (имя + ссылка на профиль)
- `Tag` — тематика и статус подборки
- `Statistic` — цена за слот, кол-во свободных мест
- `Timeline` — три точки: дедлайн заявок / дата размещения / дата публикации
- `Divider` — визуальные разделители между блоками
- `Button (type='primary')` — CTA «Подать заявку»

**Структура содержимого Drawer:**

| Блок | Содержимое |
|------|-----------|
| Шапка | Название подборки, `Tag` тематики, `Tag` статуса |
| Блогер | `Avatar`, имя, ссылка на профиль |
| Описание | Сценарий подборки — текст от блогера |
| Условия | Цена за слот (`Statistic`), свободных мест (`Statistic`), принимает ли подарки (`Tag`) |
| Категории | Допустимые категории товаров (`Tags`) |
| Даты | `Timeline`: дедлайн заявок → дата размещения → дата публикации |
| CTA | `Button primary` «Подать заявку» (`disabled` если нет мест или дедлайн прошёл) |

**Acceptance Criteria:**
- Drawer открывается без перезагрузки страницы, стандартная анимация Ant Design
- Кнопка «Подать заявку» `disabled` если нет свободных мест или дедлайн заявок прошёл
- Если у селлера уже есть активная заявка на эту подборку — кнопка заменяется на `Tag` с текущим статусом

---

### 4.3 Форма подачи заявки

**Описание:**  
После клика «Подать заявку» внутри Drawer открывается форма (смена `view` внутри той же панели, без открытия нового модала). Форма разбита на 2 шага.

**UI-компоненты Ant Design:**
- `Steps (size='small')` — Шаг 1: Условия / Шаг 2: Товары
- `Form + Form.Item` — контейнер с валидацией
- `InputNumber` — ввод цены за слот (`min = price_per_slot` подборки)
- `Switch` — «Готов подарить товар блогеру»
- `Table` с `rowSelection (type='checkbox')` — выбор НМ из арсенала
- `Input.Search` — фильтр товаров по артикулу внутри таблицы
- `Button primary` «Отправить заявку» + `Button default` «Назад»
- `Popconfirm` — подтверждение перед отправкой

**Поля формы и валидация:**

| Поле | Тип | Валидация |
|------|-----|-----------|
| Цена за слот | `InputNumber (RUB)` | Обязательное. Значение `>= price_per_slot` подборки |
| Подарить товар блогеру | `Switch (boolean)` | Необязательное. По умолчанию: `false` |
| Выбор товаров (НМ) | `Table rowSelection (UUID[])` | Обязательное. Минимум 1 товар. Только из допустимых категорий подборки |

**Acceptance Criteria:**
- Нельзя указать цену ниже минимальной — `InputNumber` выдаёт ошибку валидации
- В таблице НМ отображаются только товары из категорий, разрешённых в данной подборке
- После успешной отправки — `message.success`, статус в Drawer обновляется без перезагрузки
- Нельзя подать повторную заявку на одну и ту же подборку

---

### 4.4 Управление заявками («Мои заявки»)

**Описание:**  
Внутри вкладки «Подборки» — переключатель `Segmented`: «Все подборки» / «Мои заявки». В разделе «Мои заявки» отображаются все заявки текущего селлера с актуальными статусами.

**UI-компоненты Ant Design:**
- `Segmented` — переключатель «Все подборки / Мои заявки»
- `Table` — список заявок с сортировкой по статусу и дате
- `Tag` — визуальный статус каждой заявки (цветовая схема из раздела 3)
- `Button primary` «Подтвердить участие» — появляется при статусе «Одобрена блогером»
- `Popconfirm` — «Вы уверены, что хотите подтвердить участие?» перед confirm
- `notification` — уведомления при смене статуса

**Acceptance Criteria:**
- При одобрении блогером — появляется кнопка «Подтвердить участие» и `notification`
- После финального подтверждения — статус «Подтверждена», появляется инструкция по отправке товара
- Селлер может одновременно иметь активные заявки на несколько подборок

## 7. Принципы UI-дизайна (Ant Design)

### 7.1 Кастомизация темы

Тема настраивается через `ConfigProvider` на верхнем уровне приложения.

```tsx
<ConfigProvider
  theme={{
    token: {
      colorPrimary: '#6B4EFF', // бренд-цвет WB Инфлюенс
      borderRadius: 8,
      fontFamily: "'Inter', 'Roboto', sans-serif",
    },
  }}
>
  <App />
</ConfigProvider>
```

### 7.2 Карта компонентов по экранам

| Экран / Элемент | Ant Design компоненты |
|----------------|----------------------|
| Навигация (вкладки) | `Tabs`, `Badge` |
| Список подборок | `Table`, `Select`, `Input.Search`, `DatePicker.RangePicker`, `Tag`, `Empty`, `Skeleton` |
| Сайд-панель деталей | `Drawer`, `Descriptions`, `Avatar`, `Statistic`, `Tag`, `Timeline`, `Divider`, `Button` |
| Форма заявки | `Steps`, `Form`, `InputNumber`, `Switch`, `Table (rowSelection)`, `Popconfirm`, `Button` |
| Статусы заявок | `Tag`, `Badge`, `notification`, `message` |
| Переключатель вида | `Segmented` (Все подборки / Мои заявки) |

### 7.3 Адаптивность

Приоритет — десктоп (1440px+). Планшет (768px+) — вторичная поддержка. `Drawer` на мобильных открывается на 100% ширины. Таблицы на планшете — с горизонтальным скроллом (`scroll={{ x: true }}` в `Table`).

## 8. Безопасность

- Все API-запросы аутентифицированы через JWT-токен сессии WB Инфлюенс
- Селлер видит только свои заявки — бэкенд фильтрует по `seller_id` из токена
- Цена за слот валидируется повторно на бэкенде (защита от подмены запроса)
- Выбор товаров ограничен только теми НМ, что принадлежат текущему аккаунту
- Rate limiting на эндпоинт подачи заявок — не более 10 запросов/мин с одного аккаунта

## 10. Фазы разработки

| Фаза | Задачи | Приоритет |
|------|--------|-----------|
| **Фаза 1 — MVP** | Список подборок + фильтр по тематикам, Drawer с деталями, форма заявки (Steps), статус `pending` | Must Have |

| **Фаза 4 — Улучшения** | Фильтры по цене/датам/блогеру, сортировка в таблице, история заявок | Nice to Have |

## 11. Потенциальные сложности и решения

| Проблема | Решение |
|----------|---------|
| Конкурентные заявки на последний слот | Оптимистичный UI + серверная проверка при submit; при занятом слоте — `message.error` |
| Большой арсенал НМ (100+ товаров) | Virtual scroll в Ant Design `Table` + серверная пагинация + поиск по артикулу |
| Realtime-обновление статуса заявки | WebSocket или SSE; fallback — polling каждые 30 сек через `React Query refetchInterval` |
| Согласованность с дизайном CPO/CPP | Единый `ConfigProvider` и тема antd для всего кабинета |
| Фильтрация НМ по категориям подборки | `allowed_categories` хранится в подборке; фронтенд фильтрует таблицу НМ по этому массиву |

## Глоссарий

| Термин | Определение |
|--------|-------------|
| **Подборка** (Collection) | Тематический контент блогера, включающий несколько товаров разных селлеров |
| **Слот** | Одно место в подборке для одного селлера / товара |
| **НМ / Артикул** | Номенклатурный номер товара на Wildberries |
| **CPP** (Cost Per Post) | Оплата за публикацию поста — существующий инструмент кабинета |
| **CPO** (Cost Per Order) | Оплата за заказ — существующий инструмент кабинета |
| **Тематика** | Категориальная рубрика подборки (напр. «Спорт», «Детская одежда») |
| **Арсенал** | Совокупность НМ, которые селлер ведёт на платформе WB Инфлюенс |
| **Drawer** | Компонент Ant Design — боковая панель, выезжающая справа |
